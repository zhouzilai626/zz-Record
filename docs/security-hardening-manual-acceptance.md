# 安全加固手工验收清单

> 适用版本：本次 IPC、手机摄像头桥接与原子持久化改动。
> 自动验证已覆盖 TypeScript 编译与单元测试；以下项目必须在真实 Electron 打包或开发运行环境中手动完成。

## 0. 前置条件

- 使用一台安装 ZZ Record 的电脑和一台可接入同一 Wi-Fi 的手机。
- 电脑同时启用至少一个额外网络接口（例如 VPN、虚拟网卡或有线网络），用于确认桥接服务没有监听到非预期地址。
- 记录手机配对二维码中的电脑 LAN IP、会话参数和端口。
- 使用测试项目与测试录制目录，避免覆盖真实项目或媒体。

## 2. 首次证书引导

当前桥接仍使用局域网 HTTP 进行本地 CA 的首次引导；该路径尚未完成 token 化与证书指纹校验。因此必须在受信任的私有网络中操作。

| 编号 | 操作 | 期望结果 | 结果 |
| --- | --- | --- | --- |
| CA-01 | 使用首次连接的手机完成 CA 安装和 HTTPS 访问。 | 安装步骤可完成，HTTPS 配对页可打开，摄像头授权可用。 | ☐ 通过 / ☐ 失败 |
| CA-02 | 在非受信任公共网络或无法确认网络完整性时尝试配对。 | 不继续安装 CA；记录为不支持的高风险环境。 | ☐ 通过 / ☐ 失败 |
| CA-03 | 记录 CA 下载入口是否无须当前会话即可访问。 | 若仍可裸访问，标记为后续 `phone-camera-trust-bootstrap` 工作项，不能宣称已解决。 | ☐ 通过 / ☐ 失败 |

## 已知后续工作

- 手机 CA bootstrap 仍需要短期高熵 setup token、过期/一次性下载限制及桌面端证书指纹校验，才能降低主动局域网中间人风险。
- 当前 IPC 授权首先收敛了高危路径能力；后续应继续实施按窗口角色划分的最小 preload，以及对每个敏感 IPC 校验 sender/main frame/window role。
